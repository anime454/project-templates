## General Go file naming rules
- all lowercase
- words separated by underscores only if it improves readability
- keep names short and meaningful
- usually match the main type or responsibility

## Go package naming best practices:

- Package names should be short, all lowercase, and singular unless a plural makes sense for the abstraction.
- Avoid underscores or mixed caps.
- Package names should describe what the package provides, not necessarily what it contains.
